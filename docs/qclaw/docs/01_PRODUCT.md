# 脉刻 maik — 产品需求文档

> 本文档为 `README.md` 的产品部分，详细功能说明。

---

## 一、产品定位

**一句话：** 让每一个职场人成为人才猎手。

**使命：** 打破所有阻碍简历和人脉流动的壁垒。

**目标用户：** 华人、留学生、高学历人群，一二线城市为主。中英双语。

---

## 二、核心创新：人才伯乐体系

人才伯乐是每个用户可申请的附加身份，赋予普通职场人猎头级的推荐和变现能力。

### 2.1 伯乐等级

| 等级 | 名称 | 解锁条件 | 月推荐次数 | 简历库上限 |
|------|------|----------|-----------|-----------|
| B0 | 试用伯乐 | 注册即得 | 3次 | 10份 |
| B1 | 见习伯乐 | 成功推荐≥1次 | 10次 | 50份 |
| B2 | 正式伯乐 | 成功推荐≥5次 | 无限 | 200份 |
| B3 | 资深伯乐 | B2+认证 | 无限 | 无限 |
| B4 | 明星伯乐 | B3+top评价 | 无限 | 无限 |

**分成机制（已确认）：伯乐得60%，平台留40%。**

### 2.2 简历流动引擎

| 功能 | 来源参照 | 实现方式 |
|------|---------|---------|
| **浏览器插件采集** | 八爪盒子 | Chrome Extension MV3，猎头端一键采集简历 |
| 简历自动更新 | 八爪盒子 | 插件访问各平台，自动同步最新信息 |
| 邮箱简历录入 | 八爪盒子 | IMAP附件解析，被动沉淀候选人简历 |
| 六度人脉图谱 | 脉脉/领英 | 用户关系图谱+最短路径展示 |
| 职场问答 | 知乎 | 与招聘强关联的职场内容社区 |
| 职场小组 | 豆瓣 | 垂直社群+伯乐内推 |
| HR派单系统 | 猎聘 | HR发布需求→伯乐接单→40/60分成 |
| IM+招聘推进 | BOSS直聘 | 即时通讯+结构化推进节点 |

## 三、用户体系

```
脉刻用户（一个手机号=一个账号）
  ├── 求职者身份：简历 + 求职意向
  ├── 招聘者身份：发布职位 + 人才库
  └── 伯乐身份（可选）：简历库 + 推荐 + 变现
```

---

## 四、积分体系

| 获得 | 积分 |
|------|------|
| 成功推荐（被接受） | +100分 |
| 简历被查看 | +10分 |
| 连续30天活跃 | +50分 |

| 消耗 | 积分 |
|------|------|
| 简历库扩容（+50份/月） | -200分 |
| 悬赏单发布 | -100分 |
| 悬赏单加急 | -50分 |
| 简历曝光加速（置顶1天） | -20分 |

**B2+伯乐积分可申请提现：伯乐得60%，平台留40%。**

## 六、冷启动策略

漏斗：100万死简历 → 数据清洗 → 20万可下载简历 → 猎头/HR下载（¥5/份） → 候选人激活 → 伯乐闭环。
