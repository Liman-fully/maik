## 二、AI Agent 使用策略

### 每次开始新任务前
```markdown
# Prompt 模板
请阅读以下文档，理解脉刻项目背景：
1. README.md（项目入口）
2. docs/01_PRODUCT.md（产品需求）
3. docs/02_TECH_STACK.md（技术架构）
4. docs/03_API_SPEC.md（API规范）
5. docs/04_DB_SCHEMA.md（数据库）
6. docs/05_MEILISEARCH.md（搜索）
7. docs/07_DEPLOY.md（部署）
```

### 任务分配建议

| 任务 | 推荐Agent | 原因 |
|------|----------|------|
| Fastify API 搭建 | Cursor Agent / Claude | 复杂逻辑审查 |
| 简历解析Pipeline | Claude / GPT-4o | 需要强推理 |
| RN App开发 | Cursor + v0.dev | 页面生成+逻辑 |
| 招聘者Web | v0.dev + Cursor | 页面快速出 |
| Meilisearch索引 | 直接写脚本 | 不需要AI |
| 部署配置 | 直接写yaml/sh | 不需要AI |

## 四、每次 AI 调用的原则

**Do:**
- 每次调用前，附上相关文档路径
- 明确输出格式（TypeScript/Python/SQL等）
- 要求写出测试用例
- 要求解释关键决策

**Don't:**
- 不给上下文直接让AI写代码
- 让AI同时做多个不相关的任务
- 不检查AI输出的类型安全
- 跳过测试直接提交

---

## 五、费用控制

| 策略 | 节省比例 |
|------|---------|
| 文档一次性喂饱，减少重复解释 | 40-60% |
| 先让AI出大纲，确认后再生成代码 | 30% |
| 测试用例只在关键路径写 | 20% |
| Meilisearch/配置类不调用AI，直接写 | 固定成本 |

**总节省：预计比随机调用减少 50-70% 费用。**
