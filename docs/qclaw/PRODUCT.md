## 一、竞品分析

### 主要竞品

| 平台 | 定位 | 优势 | 缺陷 |
|------|------|------|------|
| BOSS直聘 | 直聊+算法匹配 | 响应快、用户量大 | 没有伯乐机制，猎头角色弱 |
| 猎聘 | 中高端+猎头服务 | 猎头资源丰富 | 猎头与求职者割裂，平台不透明 |
| 脉脉 | 职场社交+招聘 | 社交关系沉淀 | 社交噪音大，招聘效率低 |
| 领英 | 职业档案+社交 | 国际化、品牌强 | 国内渗透率低，功能冗余 |
| 猎萌/猎才宝 | 猎头工具平台 | 猎头工作流完整 | C端用户少，非大众产品 |

### 核心洞察

**所有竞品的共同缺陷：猎头能力是封闭的，只属于专业猎头。**

普通用户有简历、有职业人脉，但平台不给他们变现的工具。脉刻的机会点正在这里：
> **把猎头的"人脉变现"能力释放给每一个普通职场人。**

## 二、用户体系完善

### 2.1 角色定义

```
用户（统一账号体系）
  ├── 求职者（J）
  ├── 招聘者（R）
  └── 人才伯乐（B） ← 核心创新，不是独立角色，是附加身份
```

**关键逻辑：**
- 每个用户天生是 J 和 R 的混合体（职场人既找工作也招过人）
- 伯乐（B）是可申请的附加身份，按贡献等级解锁权限

### 2.2 伯乐等级体系

| 等级 | 名称 | 解锁条件 | 核心权限 |
|------|------|----------|----------|
| B0 | 试用伯乐 | 注册即得 | 推荐3次/月，简历库上限10份 |
| B1 | 见习伯乐 | 成功推荐≥1次 | 推荐10次/月，简历库50份 |
| B2 | 正式伯乐 | 成功推荐≥5次 | 推荐无限，简历库200份，可接悬赏 |
| B3 | 资深伯乐 | B2+平台认证 | 简历库无限，可发布悬赏单，可入驻伯乐广场 |
| B4 | 明星伯乐 | B3+top评价 | 专属推荐标签，优先展示，平台分成加成 |

**设计逻辑：**
- B0→B1：用第1次成功推荐建立信任，避免刷单
- B2：核心分水岭，解锁商业化权限
- B3/B4：头部用户激励机制，形成KOL效应

### 2.3 积分体系

| 积分来源 | 获得 | 消耗 |
|----------|------|------|
| 成功推荐（被接受） | +100积分 | — |
| 简历被下载 | — | -50积分 |
| 职位发布（企业） | — | -200积分/条 |
| 简历曝光加速 | — | -20积分/天 |
| 职业咨询购买 | — | 按次结算 |
| 伯乐等级维持 | B2需月≥2次推荐 | — |

**积分 ↔ 现金机制（未来可扩展）：**
- 积分不清零，可累积
- 高等级伯乐积分可兑换现金提现（平台抽成20%）
- 企业充值积分，1元=100积分

## 三、核心功能完善

### 3.1 广场（公共信息流）

**设计原则：弱化"投简历"，强化"遇见机会"**

**卡片类型：**

| 类型 | 展示字段 | 触发动作 |
|------|----------|----------|
| 求职者简历卡 | 头像、姓名、现职/现居地、核心技能标签（3个）、活跃度 | 加好友 / 立即沟通 / 收藏 |
| 招聘需求卡 | 公司logo、职位名、薪资范围、核心要求标签（3个）、发布时间 | 加好友 / 立即沟通 / 一键推荐 |
| 伯乐推荐卡 | 伯乐头像+等级标签、推荐理由、目标方向 | 查看详情 / 预约推荐 |
| 职场分享卡 | 内容摘要、互动数据 | 评论 / 转发 |

**推荐算法逻辑：**
- 招聘者看到的简历卡 = 其发布职位的匹配候选人（基于标签+地点+薪资预期）
- 伯乐看到的悬赏单 = 其简历库简历的匹配职位
- 双向匹配优先展示

**信息流排序因子：**
1. 匹配度（60%）：标签重合度
2. 活跃度（20%）：7日内登录+发布内容
3. 伯乐等级加成（10%）：B2+伯乐优先
4. 互动热度（10%）：加好友/沟通次数

### 3.2 推荐流程（核心链路）

**路径A：求职者推荐朋友**
```
求职者A → 查看招聘需求 → 发现合适候选朋友B → 
申请为伯乐（若未申请）→ 上传/选择B的简历 → 
填写推荐理由（必填，20字以上）→ 发送推荐 → 
招聘者R收到通知 → 接受/婉拒 → 
若接受：双方积分结算，推荐完成记录生成
```

**路径B：伯乐批量推荐**
```
伯乐B → 进入"我的简历库" → 选择1份简历 → 
点击"一键推荐" → 系统展示匹配职位列表 → 
选择目标职位 → 填写简短备注 → 发送 → 
招聘者R接受/婉拒
```

**路径C：招聘者反向推荐**
```
招聘者R → 在广场看到招聘需求卡片 → 
发现该职位匹配自己人才库中的简历C → 
点击"一键推荐" → 选择简历C → 发送 → 
该职位发布者接受/婉拒
```

**关键交互细节：**
- 推荐必须有明确理由（防止刷单），AI可辅助生成
- 招聘者72小时内未处理 → 自动提醒
- 婉拒时需选择原因（不合适/已找到/其他），数据用于优化匹配
- 双方均可评价对方（1-5星），评价公开，影响信用分

### 3.3 私信与沟通

**核心原则：不是"聊天"，是"推进招聘"**

- 每个推荐成功后 → 自动生成一个「招聘推进会话」
- 会话内置结构化推进节点：
  - [ ] 简历已阅
  - [ ] 初筛通过
  - [ ] 约面确认
  - [ ] Offer发出
  - [ ] 入职确认
- 会话可随时升级为普通私信

### 3.4 简历库

**伯乐简历库（核心资产）：**
- 支持从本地上传（PDF/Word）
- 支持从其他平台导入（未来扩展）
- 每份简历标注：候选人姓名（可匿名）、现职、期望职位、标签
- 简历查看权限：仅伯乐本人；分享给招聘者需对方付费下载
- 简历质量评级（AI辅助）：根据完整性+标签丰富度评级，影响推荐成功率

### 3.5 悬赏单（高级功能，B2+）

伯乐可发布悬赏：
- "寻:5年+Java后端，北京/远程，薪资60w+，成功入职酬谢2000元"
- 悬赏单可被其他伯乐接单（转推荐）

## 四、商业模式细化

### 4.1 收入来源优先级

| 优先级 | 收入项 | 定价参考 | 备注 |
|--------|--------|----------|------|
| P0 | 企业发布职位 | ¥200-800/条/月 | 按城市+职能分级 |
| P0 | 简历下载 | ¥20-100/份 | 按候选人资历分级 |
| P1 | 简历曝光加速 | ¥20/天 | 求职者付费 |
| P1 | 伯乐积分提现抽成 | 20% | 平台服务费 |
| P2 | 职业咨询 | 伯乐自定价，平台抽15% | C2C |
| P2 | 悬赏单加急 | ¥50/单 | 伯乐付费 |
| P3 | 广告位 | CPM计价 | 辅助收入 |

### 4.2 免费 vs 付费边界

**求职者免费：**
- 注册、创建简历、浏览广场、申请伯乐
- 免费曝光（按时间排序），付费加速（置顶）

**招聘者免费 vs 付费：**
- 免费：每月3条职位发布（基础版）
- 付费：解锁更多发布条数+简历库访问+数据分析

**伯乐：**
- 基础功能免费（B0）
- B2+解锁悬赏单、专属推荐标签等高级功能

## 五、信任与安全体系

### 5.1 身份认证

| 认证项 | 要求 |
|--------|------|
| 求职者 | 手机号必填；可选：LinkedIn/简历/工作邮箱认证 |
| 招聘者 | 企业邮箱认证（必填）+ 营业执照选填 |
| 伯乐 | 需完成至少1次成功推荐才能获得"认证伯乐"标签 |

### 5.2 推荐信用机制

- 每笔推荐生成「推荐档案」：推荐人→候选人→招聘者→结果→评价
- 恶意推荐（候选人投诉）→ 积分扣除 → 降级
- 招聘者恶意婉拒（超时不处理）→ 信用分下降 → 曝光降权
- 简历真实性：引入"简历验真"服务（可选付费，平台背书）

## 六、冷启动策略

### 6.1 先验证哪个点？

**推荐：先验证"招聘者愿意接受推荐"这个核心假设。**

MVP验证路径：
1. 邀请10个招聘者（HR/猎头），每人发布1个真实职位
2. 邀请20个职场人（种子用户），申请伯乐身份
3. 观察：是否有伯乐主动推荐？招聘者是否响应？
4. 验证周期：2周

**关键指标：**
- 推荐发出率（衡量伯乐意愿）
- 推荐响应率（衡量招聘者意愿）
- 推荐→入职转化率（最终价值）

### 6.2 种子用户从哪里来？

| 用户类型 | 来源 |
|----------|------|
| 招聘者 | 创始人个人人脉（HR/猎头）、LinkedIn直连 |
| 种子伯乐 | 脉脉/LinkedIn上的主动型职场人（那些经常转发职位的人） |
| 早期验证 | 在小圈子（如某个技术社群）内封闭测试 |

### 6.3 竞品差异化的一句话定位

> 脉刻：不只是找工作，是让每一个职场人都成为人才猎手。
