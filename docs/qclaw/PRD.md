## 一、产品愿景与定位

### 一句话定位

> 脉刻：让职场人脉与简历真正流动起来。

### 产品使命

打破所有阻碍简历和人脉流动的壁垒——让每一个职场人的职业网络持续被盘活，让人才推荐从"平台功能"变成"流动资产"。

### 跨时代核心：简历流动引擎

**现状：** 简历是静态的。一年更新一次，跳槽时海投，平时积灰。

**脉刻要做：** 简历是动态的。连接真实职场行为，自动更新，无需手动维护。

## 二、核心创新

### 2.1 人才伯乐体系

人才伯乐是每个用户可申请的附加身份，赋予普通职场人猎头级的推荐和变现能力。

| 等级 | 名称 | 解锁条件 | 月推荐次数 | 简历库上限 | 专属权益 |
|------|------|----------|-----------|-----------|----------|
| B0 | 试用伯乐 | 注册即得 | 3次 | 10份 | — |
| B1 | 见习伯乐 | 成功推荐≥1次 | 10次 | 50份 | — |
| B2 | 正式伯乐 | 成功推荐≥5次 | 无限 | 200份 | 可发悬赏、可接单 |
| B3 | 资深伯乐 | B2+认证 | 无限 | 无限 | 可入驻伯乐广场展示 |
| B4 | 明星伯乐 | B3+top评价 | 无限 | 无限 | 专属标签、优先展示、**分成加成** |

**分成机制（已确认）：伯乐得60%，平台留40%**

### 2.2 简历流动引擎（核心差异化）

让简历从"静态档案"升级为"实时流动资产"的完整功能矩阵：

## 三、简历流动引擎 — 功能详设

### 3.1 简历自动更新

**原理：** 用户授权脉刻连接其已有的招聘平台账号，脉刻定期拉取最新信息，自动更新简历档案。

**支持接入的平台（按优先级）：**

| 平台 | 接入方式 | 数据字段 | 优先级 |
|------|----------|----------|--------|
| BOSS直聘 | 模拟登录+API（需用户Cookie授权） | 职位、薪资、状态 | P0 |
| 猎聘 | Cookie授权 | 简历主体、工作经历 | P0 |
| 领英 | OAuth官方授权 | 工作经历、技能、教育 | P0 |
| 前程无忧 | Cookie授权 | 简历完整字段 | P1 |
| 智联招聘 | Cookie授权 | 简历完整字段 | P1 |
| 脉脉 | OAuth授权 | 工作经历、人脉数据 | P2 |

**更新频率：** 用户自定义（实时/每日/每周），超出配额需积分兑换

**隐私控制：**
- 自动更新范围由用户设置（可关闭任一平台的数据同步）
- 仅同步用户明确授权的字段
- 同步记录可查可删除

**用户收益：**
- 简历永远是最新的，无需手动维护
- 一处更新，全平台同步
- 简历完整度评分提升，吸引更多伯乐推荐

---

### 3.2 邮箱简历录入

**场景：** 用户邮箱里有猎头发来的候选人简历附件（PDF/Word），一键入库。

**功能流程：**

```
用户授权邮箱（IMAP/SMTP）→
脉刻扫描邮件附件（PDF/Word）→ 
AI解析简历结构化字段（姓名/公司/职位/邮箱/手机）→
自动入库简历库，同时发送通知告知用户：
"已从邮箱录入3份简历，是否确认添加到您的简历库？"
```

**支持邮箱：**
- Gmail（IMAP原生支持）
- QQ邮箱 / 163邮箱 / 企业邮箱（IMAP）
- Outlook

**隐私说明：** 仅扫描含简历关键词的邮件附件，不读取其他内容；授权随时可撤销。

**用户收益：**
- 猎头/HR日常收到的简历自然沉淀为个人简历库
- 无需手动整理，被动积累伯乐资产

---

### 3.3 六度人脉图谱

**定义：** 任意两个脉刻用户之间，展示最短的连接路径（共同好友链）。

**交互场景：**

> "我想联系某公司CTO，但不认识他"
> → 脉刻展示：「你 → 张三（同事）→ 李四（前同事）→ CTO」
> → 点击任意节点可直接发起好友请求或托人引荐

**数据来源：**
- 脉刻站内好友关系（原生数据）
- 用户授权导入的通讯录（手机号匹配）
- 简历中填写的同事信息（经授权）

**显示层级：**
- 1度（直接好友）：可立即沟通
- 2–3度：显示路径，可申请托人引荐
- 4–6度：显示路径，不可引荐（防止骚扰）
- 无连接：提示"暂无共同好友"

**隐私控制：**
- 用户可隐藏自己的人脉关系（"仅自己可见"）
- 好友申请通知：被托人引荐者收到通知，可选择接受/拒绝

---

### 3.4 问答社区

**定位：** 不是知乎的复制，是**与职业机会强关联的职场问答**。

**核心机制：**
- 用户可发起职场问答（行业选择/职能选择/城市）
- 回答者可关联自己的简历（展示专业背景）
- 优质回答者的简历可被招聘者直接查看并发起邀请
- 问答内容可打赏（伯乐积分）

**内容与招聘的连接：**
```
用户A发布问题："35岁转AI算法工程师，可行吗？"
↓
猎头/招聘者浏览相关问答，发现用户A背景合适
↓
点击A的简历卡片 → 发起"我对您的背景感兴趣"
↓
用户A收到伯乐邀请 → 进入推荐流程
```

**内容治理：**
- 发布需伯乐身份（B1+）
- 禁止个人隐私（薪资/具体公司名）
- 举报机制完善

---

### 3.5 职场小组

**定位：** 按行业/城市/职能/阶段组建的垂直社群。

**小组类型：**

| 类型 | 示例 | 内容 |
|------|------|------|
| 行业组 | 互联网产品经理、北京Java开发 | 行业动态、职位内推 |
| 阶段组 | 应届生求职、30岁转行 | 经验分享、互帮互助 |
| 兴趣组 | 远程工作、创业中 | 资源共享、协作 |
| 伯乐专属组 | B2+认证伯乐交流群 | 悬赏接单、技巧分享 |

**核心功能：**
- 小组内可发职位（免费，但有数量限制）
- 小组内可推荐伯乐简历（免费）
- 小组管理员（B2+可申请创建）有内容审核权

---

### 3.6 HR派单系统（猎聘式）

**场景：** HR/企业有批量招聘需求，直接派发给平台伯乐执行。

**运作流程：**

```
HR在脉刻发布"批量招聘需求"
↓
平台展示给符合方向的伯乐（B2+）
↓
伯乐接单并在自己简历库中匹配合适候选人
↓
伯乐发起推荐 → HR接受 → 推荐成功
↓
分成：HR付费（平台代收）→ 伯乐得60%，平台留40%
```

**派单类型：**
| 类型 | 说明 | 结算方式 |
|------|------|----------|
| 单职位悬赏 | HR指定职位，伯乐独立完成推荐 | 按成功入职结算 |
| 批量RPO | HR批量需求，伯乐批量交付 | 按交付数量阶梯结算 |
| 紧急加急 | HR急招，伯乐快速响应 | 加急费（¥100–300）|

**HR收益：** 无需自己在海量简历中筛选，由专业伯乐做第一轮过滤，效率大幅提升。

---

### 3.7 即时通讯 + 招聘推进

**定位：** 不是通用IM，是专为招聘场景优化的推进工具。

**消息类型：**

| 类型 | 说明 |
|------|------|
| 自由对话 | 伯乐与招聘者/求职者的普通聊天 |
| 推荐会话 | 推荐成功后自动生成的结构化会话 |
| 约面通知 | 系统触发，告知面试安排 |
| 系统通知 | 积分变动、推荐状态变化等 |

**招聘推进会话节点：**

```
[简历已阅] → [初筛通过] → [面试邀请已发] → [面试完成] → [Offer确认] → [入职到岗]
```

每完成一个节点，系统自动记录时间戳，作为推荐档案数据。

## 七、冷启动策略（核心资产）

> **已有100万份死简历，这是脉刻最大的冷启动杠杆。**

### 7.1 简历库存策略

| 阶段 | 行为 | 收益 |
|------|------|------|
| 启动期 | 猎头入驻×100家，各下载100份 | 100家×100份×5元 = **5万元即时收入** |
| 成长期 | 500家猎头，月下载2000份 | 500×2000×5 = **50万/月** |
| 成熟期 | 下载→候选人激活→伯乐推荐→分成 | 平台40%持续抽成 |

### 7.2 简历激活漏斗

```
100万份死简历（沉睡资产）
    ↓ 数据清洗+结构化（去重、补全字段、分类标签）
20万份可下载简历（质量达标）
    ↓ 猎头/HR入驻并下载（5元/份）
    ↓ 平台通知候选人："您的简历被某猎头查看，是否接收推荐？"
    ↓ 候选人同意 → 自动申请伯乐 → 活用户激活
伯乐正式激活 → 推荐生态闭环
```

### 7.3 猎头/HR激活路径

1. **邀请入驻**：主攻中小猎头公司（无自建简历库）和中小企业HR（无招聘预算买会员）
2. **卖点**：5元下载全网简历库，远低于猎聘/前程无忧
3. **留存**：猎头下载后自然留在平台，持续使用推荐功能

### 7.4 核心验证假设

- 猎头/HR 是否愿意付费5元下载一份简历？
- 下载后是否会留在平台上持续使用？

## 十、技术架构（已确认）

### 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| **前端** | React Native + Expo | 一套TypeScript → Web+iOS+Android三端 |
| **后端** | Node.js + Express/Fastify | 轻量API服务 |
| **P** — 关系数据库 | PostgreSQL（腾讯云） | 用户/伯乐/职位/订单/积分 |
| **M** — 搜索引擎 | Meilisearch（自建，内存限制0.5） | 百万简历全文检索+中文分词 |
| **T** — 缓存/队列 | 腾讯云Redis（轻量版） | 会话/队列/积分计算缓存 |
| 文件存储 | 腾讯云COS（已有） | 简历附件/头像/原始文件 |
| 简历解析 | PaddleOCR + Marker/MarkItDown + 阿里NLP + DeepSeek | 见简历解析pipeline |
| 中间数据格式 | JSON（标准化简历档案） | 入库前统一兜底校验 |
| 进程管理 | pm2（需配置内存限制） | 服务器直接装，不走Docker |

### 部署注意事项

> ⚠️ 2核4G服务器同时跑Meilisearch+Node服务+Redis会OOM。
> **解决方案：**
> - Meilisearch 启动加 `--max-memory-ratio 0.5`
> - pm2 配置 `max_memory_restart: "512M"`
> - 最终建议：升级到4核8G，资源问题彻底消除

### 简历解析 Pipeline

```
输入（图片/PDF/Word/Excel/网页/邮件附件）
    ↓
格式判断 → 对应解析器
    ↓
PaddleOCR（图片/扫描件）→ Markdown（复杂排版）→ 纯文本
    ↓
正则规则 + 阿里开源NLP模型（结构化提取）
    ↓
DeepSeek（兜底补全 + 标签生成 + 质量评分）
    ↓
JSON标准化档案（统一Schema + 校验）
    ↓
PostgreSQL（结构化字段） + Meilisearch（全文索引）
```

### 数据存储架构

```
腾讯云 COS（原始文件）
    ↓
简历解析 Pipeline（2核4G服务器）
    ↓
PostgreSQL（结构化数据 + 业务逻辑）
Meilisearch（简历全文索引，搜索用）
Redis（缓存 + 积分队列 + 会话）
```
